Creating a new OIOSAML .Net release

This describes how to create a new release for OIOSAML .Net

Building the packages

  • Remember to increment the version number of the assemblies
  • Remember to add the newest version to the list of releases in the documentation
  • Build the project in release mode
  • Build the installer project

Build the zip package

Uploading the install package to digitaliser.dk

  • Login on digitaliser.dk and go to the newest version of the ressource (should be http://digitaliser.dk/resource/2513426)
  • Choose Funktioner and click on "Opret ny version"
  • Change the metadata of the ressource accordingly
  • Delete the old files and upload the new files of the release (zip file + documentation)
  • Publish the new version when ready

Change the frontpage of the group

Create and publish nuget packages

This will create new version of the packages and publish them to nuget

  • Change the release note in the .nuspec file
  • Pack the dk.nita.saml20 project: nuget pack dk.nita.saml20.csproj -Symbols -Properties Configuration=Release
  • Push the package: nuget push pakkenavn
  • Pack the dk.nita.saml20 project: nuget pack dk.nita.saml20.ext.audit.log4net -Symbols -Properties Configuration=Release
  • Push the package: nuget push pakkenavn
